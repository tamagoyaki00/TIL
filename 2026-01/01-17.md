## Javaの勉強

### オーバーロード

引数の型や数が違えば同じ名前でメソッドを複数定義できる

```ruby
	void setData(Sring n) {
		name = n;
	}
	void setData(String n, int e, int m) {
		name = n;
		engScore = e;
		mathScore = m;
	}
```

## コンストラクタ

オブジェクトの生成（インスタンスの生成）とともに自動的に呼び出される特殊なメソッドのこと。オブジェクトの初期化のために使われる。

ルール①　メソッド名がクラス名と同じ名前　※メソッド名は通常小文字だが、クラス名と同じなので大文字

ルール②　戻り値を持たない

ルール③　newクラス名（コンストラクタへの引数）

コンストラクタを定義していない場合は、自動でデフォルトコンストラクタ（引数・処理なし）が生成

例：

```ruby
public class Student {
	String name;
	int engScore;
	int mathScore;
	
	Student(String n) {
		name = n;
	}
	Student(String n, int e, int m) {
		name = n;
		engScore = e;
		mathScore = m;
	}
	void setScore(int e, int m) {
		engScore = e;
		mathScore = m;
	}
	
	void display () {
		System.out.println(name + "さん");
		System.out.println("英語" + engScore + "点・数学" + mathScore + "点");
	}
```

```ruby
public class StuSample {

	public static void main(String[] args) {
		Student stu1 = new Student("菅原");
		Student stu2 = new Student("村山",75,100);

		stu1.setScore(90,80);
		stu1.display();
		stu2.display();
		
	}
}
```

※コンストラクタを生成したら、初期値をいれないとエラーになるので注意

Student stu1 = new Student();だとエラーになる

## static変数、staticメソッド

全インスタンス変数が使えるメンバ変数やメソッドを定義するにはstaticを指定する。

オブジェクトを生成しなくても利用できる。

`クラス名.変数(メソッド)名`

例:

```ruby
class Student {
	String name;
	int counter = 0;
	
	Student(String n){
		name = n;
		counter++;
		System.out.println(name);
	}
	void display() {
		System.out.println(counter);
	}
}	
	
//staticを使った場合
class Student {
	String name;
	static int counter = 0;
	
	Student(String n){
		name = n;
		counter++;
		System.out.println(name);
	}
	static void display() {
		System.out.println(counter);
	}
}	
```

```ruby
class StuSample {
	public static void main(String[] args){
		Student stui1 = new Student("田中");
		stu1.display();
	
		Student stu2 = new Student("佐藤")
		stu2.display();
}
	
	//出力　田中 1 佐藤 1
	
	
// staticを使った場合
class StuSample {
	public static void main(String[] args){
		Student stui1 = new Student("田中");
		Student.display();
	
		Student stu2 = new Student("佐藤")
		Student.display();
}

//出力　田中 1 佐藤 2
```

このようにインスタンスを２つ作っても、staticの部分は共通しているので、2回目のdisplayを呼び出したときには2と表示される。

また、インスタンスを生成しなくてもstaticは呼ぶことができるので、

`Student stui1 = new Student("田中");`の前に`Student.display();`を呼ぶと0と表示される。
