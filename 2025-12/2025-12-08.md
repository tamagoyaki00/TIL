### propsとchildren
親⇒子に渡す

例： propsを使って親から子へデータ（name）を渡す
```
#子コンポーネント
import React from "react" ;

export function Message(props) {
    return <p>こんにちは、{props.name}さん！</p>;
}
```

```
#親コンポーネント
export default function App (){
    return (
        <Message name = "たまごやき" />
    );
}
```


childrenは特別なpropsで、コンポーネントのタグに挟まれた内容を指す。


```
#子コンポーネント(枠だけ用意する)

function BentoBox({ children }) {
    return (
        <div>
            今日のお弁当:
            <span>{children}</span>
        </div>
    );
}

```

```
#親コンポーネント（中身を渡す）

function App

function App() {
  return (
    <BentoBox>
      おにぎり
      からあげ
      ブロッコリー
    </BentoBox>
  );
}

```

- `props` は「**パーツの設定値**（たとえばラベル、色、サイズ）」
- `children` は「**中に何を入れるか**（コンテンツ全体を渡す）」

### ルーティング
Turbo-Frameのように、画面全体をレンダリングするのではなく、画面の一部を差し替えるもの。

`pm i react-router-dom` でReact Routerをインストールする

```
import { BrowserRouter, Routes, Route, Link } from "react-router-dom";
import Home from "./Home";
import About from "./About";

function App() {
  return (
    <BrowserRouter>
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/about" element={<About />} />
      </Routes>
    </BrowserRouter>
  );
}

```

`BrowserRouter` で全体をラップする。ルーティング機能が有効になる  
`Routes` は、複数の<Route>の中から、現在のURLに一致する最初のルートのみをレンダリングします  
`Route` は、URLパスと対応するコンポーネントを定義します。  

Railsではルーティングに記述することで自動でヘルパーパスが生成されていたが、Reactでは自分で定義する。

```
<Link to="/">ホーム</Link>

<Link to="/about">サイトについて</Link>
```
などLink toに上記で定義したパスを指定することで、対応するコンポーネントを表示させることができる。
