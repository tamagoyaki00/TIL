## チェリー本
チェリー本を読んでいて、正規表現についてあまり知らないなぁと感じたので、
こちらの記事を読んでみた。一部抜粋
https://qiita.com/jnchito/items/893c887fbf19e17d3ff9

特別な文字を 「メタ文字」と呼ぶ

`\d` は半角数字を意味する
文字の個数を限定するときは {n,m} や {n} というメタ文字を使います（文字量を指定するので、特に 「量指定子」 と呼ばれる）。
`{n,m}` は「直前の文字が n 個以上、m 個以下」の意味。
`{n}` は「ちょうど n 文字」の意味。
`[ABC]`はABCのいずれか一文字という意味
`[0-9]`これは0～9の意味でハイフンが含まれるという意味ではないので注意。ちなみに[0-9]と\dは同じ意味

```
# 半角数字2～5文字　ハイフン　半角数字1～4文字　ハイフン　半角数字4文字
\d{2,5}-\d{1,4}-\d{4}

# 090-0000-0000という番号もあれば090(0000)0000どちらにも一致させる場合
\d{3}[-(]\d{4}[-)]\d{4}
```

`?`「～が1文字、または無し」の意味
`.`任意の一文字の意味
`.+`直前の文字が1文字以上
`.*`直前の文字が0文字以上

`\w`は英単語を校正する文字の意味。小文字だけではなく、大文字も含む
`[^A]`はA以外の文字を意味する。

呪文だった、正規表現がようやく理解できた！

## Paiza Cランク
一行に、文字列と数字の組み合わせの問題が初めてでた。
その時は、以下のようにまず文字列として取り出してから数値に変換させる。
```
hoge = []
n.times do
  a, i = gets.split
  hoge << [a, i.to_i]
end
```
