- クラスを理解しよう
Bランクの練習問題を行ったときに、クラスの問題が出てきた。Railsがよしなにやってくれている部分でもあるのであまり理解していなかったので、復習。

クラス変数:　クラス変数を宣言したクラスのインスタンス全てで共有して利用できる変数。「@@」を先頭につける

例：
```
class Item
    @@tax = 1.1

    def initialize(price, quantity)
        @price = price
        @quantity = quantity
    end

    def total()
        (@price * @quantity * @@tax).round
    end
end

apple = Item.new(120, 15)
total = apple.total()
puts "合計金額は#{total}円です。"

orange = Item.new(85, 32)
puts "合計金額は#{orange.total}円です。"
```

### 例外処理の機能
- begin あらかじめコードを指定して、プログラム実行時に、処理の問題を検出
- rescue 問題を検出したとき、どのように対応するか記述
- raise 対応を記述していない場合、メソッドの呼び出し元に対応を任せる
- ensure 例外が発生してもしなくても実行される処理

Exceptionはスーパークラスなので、一番初めに書くと、すべての処理を捕まえる。
より具体的なエラー`ZeroDivisionError`などと一緒に書くのであれば、具体的なものを先に書く。


## React
ReactではRailsのようにファイルの置き場所が決まっていない。  
いろいろあるみたいだが、個人開発ではファイル別がわかりやすそう。Youtubeで紹介されていたReactの練習でもこの方式だったので、初学者にとってもわかりやすい。  
例：
```
src/
  components/   # UI部品
  pages/        # ページ単位
  hooks/        # カスタムフック
  services/     # API通信
  utils/        # 共通関数
  assets/       # 画像やフォント
  styles/       # CSSやテーマ

```

大規模になったり、複数人の作業になるのであれば、フィーチャーベースが分かりやすそう。  
フィーチャーベースとは、アプリケーションの各機能ごとにフォルダを作成し、その機能に関連するすべてのコード（コンポーネント、フック、コンテキスト、スタイル、テスト、サービスなど）を1つのフォルダにまとめて管理する方法  
例：
```
src/
  features/
    auth/
      components/
      hooks/
      services/
    product/
      components/
      pages/
      services/
  shared/
    components/
    hooks/
    utils/


```

その他にもAtomic Designなどあるみたいなので、チームによって設計がちがうことを覚えておく。
